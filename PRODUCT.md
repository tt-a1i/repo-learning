# Product

## Users

开发者与新贡献者第一次接触陌生仓库时使用：需要尽快知道**它为什么值得关心**，建立可信的架构与运转心智模型——不是抠实现细节。

## Purpose

把一个仓库地址变成**一份**可离线打开的精致学习页。用源码证据讲清：

- **项目价值**（为谁、解决什么、相对替代方案强在哪）
- **架构与脉络原理**（怎么转、为何长成这样）
- **一条主流程 + 少量关键概念**

可选附「若要继续读码」的入口提示。不是 README 换皮，不是审计报告，不是源码导读手册。

## Personality

冷静、精确，像一份可印刷的深度简报。气质对齐 **浅色编辑部研报**：纸色底、青绿强调、衬线标题、发丝线分区、Mermaid 嵌在浅底 panel。

## Principles

1. 首屏立刻回答「这是什么」——项目名是 masthead 级信号；紧接着回答「为什么重要」。
2. 呈现停在职责、因果与取舍；调查可以细，页面默认不堆行号与函数清单。
3. 强层级靠排版与发丝线，不为了「干净」而发灰发平，也不靠暗色发光堆氛围。
4. 图与文字相邻、互相印证；图板融入纸色主题。
5. 自顶向下；少而准。大仓先讲主脊柱，不追求覆盖率。
6. 单文件交付；模板负责气质，Agent 负责理解。

## Anti-goals

- JSON schema / 多层校验流水线作为主路径
- 无样式「语义站」冒充产品
- 无证据的示意图与空洞指标
- 文件树、依赖清单、调用链冒充架构理解
- AI 营销风（紫渐变、毛玻璃卡片墙、三列概念卡）
- 发灰发平的「说明书默认皮」
- 夜间霓虹 / 铜光发光图鉴冒充研报
