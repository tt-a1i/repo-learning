# Report Design

Offline-first HTML learning brief. No CDN, no remote fonts, no analytics.

## Language

- **Default: 简体中文** for both agent-authored JSON content and HTML chrome.
- English UI only when the user asks, or when `--lang en` / `meta.locale: "en"` is set.
- Keep evidence paths, ids, and code identifiers in their original form.

## Product intent

This is a **reading brief**, not a monitoring dashboard and not a stack of white cards.

- Full-bleed cover masthead → sticky horizontal chapter index → numbered chapter stream.
- Lead with one thesis; stats are a quiet index under the cover.
- Modules/risks/flows read as lists and columns, not SaaS cards.
- Diagrams sit in occasional full-bleed bands; most content stays on the page ground.
- Collapse visual weight when a section has little data (`is-empty`), but keep the eight required section ids for validation.

## Required sections (DOM ids)

Each must appear exactly once as `section id="..."`:

| id | Purpose |
|----|---------|
| `overview` | Thesis, claims, language chart |
| `topology` | Module map SVG + module cards |
| `dependencies` | Swimlane SVG + external list |
| `flows` | Critical runtime flows |
| `risks` | Dense matrix + risk cards |
| `tests` | CI/test notes + matrix table |
| `roadmap` | Contributor learning path (metro timeline) |
| `appendix` | Gaps, open questions, files examined |

Also required: `script id="report-data" type="application/json"` with full v1 JSON (escape `</script>` as `<\/script>`).

**Validation chain:** JSON input → `schema_validate.collect_errors` in `generate_report.py`; shipped HTML → same `collect_errors(..., strict=True)` on embedded JSON in `validate_report.py --strict`, plus section-id uniqueness and offline pattern checks.

## Visual system

- Editorial stacks (offline): serif for titles/claims (`Iowan Old Style` / Palatino / Georgia), sans for UI (`Avenir Next` / Segoe UI), mono for paths/evidence.
- Cool gray page ground (`#f3f5f7`) with indigo accent (`#2546a8`); dark via `prefers-color-scheme`. Avoid teal dashboards, purple gradients, and cream+terracotta clichés — accent lives in type/rules, not card chrome.
- No panel cards as the primary layout unit. Chapters are stream sections with `01`–`08` numbers and a one-line dek.
- Accent hue still derived from `meta.repo_name` + `meta.repo_path` (`--hue`) for per-repo cover variation.
- Evidence: chips + drawer; confidence as mono uppercase meta.

## Cover + chapter stream

1. **Cover** — full-bleed hero (not a bordered card): kicker, large serif title, thesis, meta pills, quiet KPI index.
2. **Sticky TOC** — horizontal numbered index (`01 Overview` …), not a left rail.
3. **Chapters** — each required section is a `section.chapter` with number + title + dek; body uses splits/lists/bands as needed.

## Legends, empty states & caps

- Core charts keep compact legends (module kind, edge type, risk severity, test status).
- Empty data renders a named empty state, not a blank SVG.
- Sections with no useful payload get `class="panel is-empty"` (still present for id contract).
- Caps: modules 24, risks 30, tests 24, roadmap 12, external 12, with `+N more …`.

## Claim / card rendering

- Claim labels resolve `text|title|summary|description|name|label` — never emit empty `<strong></strong>`.
- Topology lists modules as cards (name, kind, description, evidence), not evidence-only bullets.
- Risks render as severity-sorted cards; matrix only shows occupied rows/columns.
- Flows prefer `title|name|id` and optional `summary`; steps are numbered rows.

## Module cross-highlight & filters (lightweight JS)

Nodes, edges, risk cells, claim items, test rows carry `data-module` / `data-kind` / `data-severity` / `data-status` / `data-category`. Click topology/dependency node or risk cell to highlight; click again to clear. Filter chips (risk severity) toggle via classes only — no `innerHTML`, no `eval`. Without JS, content stays readable.

## Chart modules

Generated by `scripts/generate_report.py`:

1. **Language bars** — track + fill, right-aligned percent
2. **Topology** — readable grid (kind subtitle, truncated labels), clickable nodes
3. **Dependencies** — swimlane by occupied kinds only (bugfix: no duplicate empty Other lanes); cubic edges + arrows; external pills in data lane
4. **Risk matrix** — only modules/categories that have risks; count + severity symbol; unknown `module_id` → `global/unknown`
5. **Test matrix** — HTML table, missing/unknown first; ✓/✗/?
6. **Roadmap** — metro timeline with evidence chips

Prefer SVG over Mermaid. No force graph, pan/zoom, drag, D3, or Mermaid rendering. Mermaid may appear only as **source text** in appendix.

## Navigation: sticky TOC

`generate_report.py` emits `<nav id="toc" class="toc-rail">` as a **top sticky horizontal index** (`<ol>` of numbered anchors).

- **No JS** — plain `#overview` links.
- **With JS** — `IntersectionObserver` marks the active section.
- Never a left sidebar rail; mobile keeps the same horizontal pattern.

## Evidence chips & drawer

Evidence is a chip (`<button class="ev-chip">`) with `data-evidence`, `data-claim`, `data-section` (HTML-escaped). Global `<dialog id="ev-drawer">` fills via `textContent` on click.

- Without JS, chip text stays visible.
- List evidence joined with `; `.
- Escape with `html.escape(..., quote=True)`.

## Offline link validation

`validate_report.py` flags remote/executable references in markup/CSS only (`src`/`href` / CSS `url(...)` beginning with `http(s)://`, `//`, or `data:`). Plain URLs inside embedded JSON or evidence text are allowed.

## Security & privacy

- HTML-escape all dynamic text
- Do not embed secret values — key names / file locations only
- No executable remote resources
- Drawer uses `textContent`, never `innerHTML`

## Print, screenshot & mobile

- `@media print` — high contrast, hide rail/filters/chips/toggle
- `.screenshot-mode` — hide rail/filters/toggle
- Mobile — horizontal TOC; charts/tables scroll horizontally

## Template reference

See `assets/report-template.html` for structural reference. Production reports are emitted by `generate_report.py` into `index.html`.

## Optional enhancement (non-default)

If the user explicitly requests richer diagrams **and** allows tooling, `$archify` may produce supplementary HTML — keep the main report offline.
