# Product

## Register

brand

## Users

开发者、技术负责人和新贡献者在第一次接触陌生仓库时使用它。他们需要在短时间内建立可信的系统心智模型，并知道下一步该从哪些源码入口开始。

## Product Purpose

把一个仓库地址转化为一份可离线打开的学习网站。网站用源码证据解释项目的目标、模块关系、真实流程、关键概念和学习顺序，而不是把 README 重新排版。

## Brand Personality

冷静、精确、可亲近。它像一张制作精良的产品发布页，也像一份可靠的工程地图：先让人看懂，再邀请人深入。

## Anti-references

- 以巨大空白、巨型标题或无意义动效伪装高级感的页面。
- 把每一段内容都塞进同款圆角卡片的 AI 营销模板。
- 没有源码依据的指标、示意图或产品承诺。
- 依赖 CDN、远程字体或外链图片才能成立的展示页。

## Design Principles

1. 首屏马上回答项目是什么，并让架构主视觉与文字在同一阅读节奏中出现。
2. 让证据成为交互的一部分，而不是藏在附录里。
3. 每种知识结构使用适合自己的视觉语法：图、流程、代码入口和学习路径不共享一套卡片皮肤。
4. 留白服务于层级和阅读，不服务于制造距离感。
5. 保持离线、响应式、可访问和低动效容错，任何脚本失效都不应隐藏内容。

## Accessibility & Inclusion

满足 WCAG AA 的文字和交互对比度；保持键盘可达、可见焦点、44px 触控目标、深浅主题和 reduced-motion 降级；在窄屏上将复杂布局收束为单列或局部横向滚动，避免整页横向溢出。
